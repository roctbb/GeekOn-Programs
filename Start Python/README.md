## Критерии оценки теста по теории:
0 - 5 баллов в зависимости от % выполнения

## Критерии оценки ДЗ для типовых задач:
- 10 - поставленная задача выполнена полностью и без ошибок, код проходит проверку на PEP-8, разделен на логические части, повторяющиеся инструкции оформлены в виде функций
- 9 - Незначительные ошибки при валидации PEP-8
- 8 - Код не проходит валидации PEP-8
- 7 - Повторяющиеся наборы инструкций в коде, логические блоки программы не выделены в функции
- 6 - При некоторых специфичных наборах входных данных программа выдает неправильный результат или завершается с ошибкой
- 5 - В большинстве случаев программа выдает неправильный результат или завершается с ошибкой
- 1 - 4 - Поставленная задача выполнена неполностью
- 0 - Код содержит синтаксические ошибки/программа не запускается

**Для проектных задач при оценке выше 5:**

- + 1 - 5 баллов за использование внешних ресурсов не перечисленных на занятии
- + 1 - 5 баллов за использование внешних ресурсов

Итоговая оценка для проектных задач умножается на два.
